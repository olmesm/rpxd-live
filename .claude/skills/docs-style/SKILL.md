# rpxd docs style

Before writing or editing user-facing markdown, read
[`docs/style-guide.md`](../../../docs/style-guide.md) and follow it. It is the
single source of truth for docs voice; do not restate or fork its rules here.

The short version, for orientation only:

- Open every page with the reader's goal in plain English.
- Short sentences, one idea each; no §-refs, issue numbers, or source paths in
  user docs; gloss insider jargon at first use.
- Package READMEs follow the fixed shape defined in the guide (plain first
  sentence → install line → audience-routing sentence → content → docs link).
- Keep pages equal or shorter; keep the honest caveats; don't blandify.

Internal documents (`spec.md`, `CLAUDE.md`, `docs/adr/`, `docs/reviews/`) are
exempt.
