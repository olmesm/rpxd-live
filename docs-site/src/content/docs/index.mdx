## What is rpxd?

rpxd is a live-object framework for React. You write a **live object** per
page — a server-side, per-session stateful object with a `setup`, reducers
(`rpc`s), and a `render` (plus `load` for URL-dependent data). State changes are streamed to the browser as Immer
patches over SSE or WebSocket, and the client replays your optimistic updates
on top for instant feedback.

The whole contract is inferred from a single fluent chain — no manual
annotations, no generated client, no glue.

## One page, one object

Here is a multiplayer project board — the whole page. `setup` subscribes to
the org channel and declares the state shape; `load` fills it from your
database; the `create` rpc writes, then broadcasts to every other session; `on`
applies that broadcast; `render` is plain React. The state shape, the `create`
payload type, and the client `rpc.create` facade are all inferred from this one
chain — there is no API layer, no client store, no cache to keep in sync.

```tsx
// routes/org.$orgId.board.tsx — one live object per page, fully inferred
export default live("/org/$orgId/board")
  .setup((ctx) => {
    ctx.subscribe(`org:${ctx.params.orgId}`);
    return { projects: [] as Project[] }; // sync skeleton — data loads in `load`
  })
  .load(async (_url, ctx) => {
    const projects = await db.project.findMany({ where: { orgId: ctx.params.orgId } });
    ctx.patchState((s) => { s.projects = projects; });
  })
  .rpc("create", (r) =>
    r.input(z.object({ name: z.string() })).handler(async ({ name }, ctx) => {
      const p = await db.project.create({ data: { orgId: ctx.params.orgId, name } });
      ctx.patchState((s) => { s.projects.push(p); });
      ctx.broadcast(`org:${ctx.params.orgId}`, "project.created", p);
    }),
  )
  .on("project.created", (state, p) => { state.projects.push(p); })
  .render(({ state, rpc, sync, keyOf }) => (
    <ul>{state.projects.map((p) => <li key={keyOf(p.id)}>{p.name}</li>)}</ul>
  ));
```

## A token stream is O(delta) on the wire

Because string-suffix growth diffs to an `append` op that carries only the new
text, an LLM loop that appends tokens streams the *delta*, not the whole answer
so far — the wire cost is O(delta) per token, not O(total). Streaming is just a
`for await` loop over `ctx.patchState`:

```ts
.rpc("ask", (r) =>
  r.input(z.object({ prompt: z.string() })).handler(async ({ prompt }, ctx) => {
    ctx.patchState((s) => { s.answer = ""; s.thinking = true; });
    for await (const delta of llm.stream(prompt, { signal: ctx.signal })) {
      ctx.patchState((s) => { s.answer += delta; }); // → append op, O(delta)
    }
    ctx.patchState((s) => { s.thinking = false; });
  }),
);
```
